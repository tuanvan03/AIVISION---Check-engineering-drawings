# Báo Cáo Dự Án AI Vision Drawing Checker
**Ngày:** 02/05/2026

---

## 1. Tổng Quan Dự Án

**AI Vision Drawing Checker** là hệ thống kiểm tra bản vẽ kỹ thuật CAD tự động sử dụng AI, giúp kỹ sư phát hiện lỗi về kích thước, chú thích và vi phạm tiêu chuẩn ISO/ASME mà không cần đối chiếu thủ công với tài liệu tiêu chuẩn.

---

## 2. Luồng Hoạt Động Hiện Tại

```
1. User upload file DXF
   ↓
2. DXF Parser → JSON + SVG/PNG
   ↓
3. Drawing Type Predictor → đề xuất loại bản vẽ
   ↓
4. User xác nhận loại bản vẽ qua chat
   ↓
5. Analysis Node → GPT-4o phân tích tổng thể
   ↓
6. Review Node → 3 checker tuần tự (trả Structured JSON)
   ├─ Dimension Checker (RAG query ISO 129)
   ├─ Annotation Checker (RAG query ISO 7200)
   └─ Standard Checker (RAG query ISO 1101/128)
   ↓
7. Self-Check Node (nếu entity count ≥ 200)
   ├─ Tra entity_handles → tính bbox
   ├─ Crop vùng lỗi từ PNG
   ├─ GPT-4o xác nhận từng lỗi high/medium
   └─ Loại false positive
   ↓
8. Render báo cáo 5 phần bằng tiếng Việt
   ↓
9. User hỏi đáp về tiêu chuẩn (RAG Chat Node)
```

## 3. Cơ Sở Dữ Liệu

### 3.1. ChromaDB (Vector Database)

**Mục đích:** Lưu trữ embeddings từ 7 tài liệu ISO cho RAG retrieval

**Cấu hình:**
- **Thư mục:** `.chroma_db/`
- **Collection:** `iso_standards`
- **Embedding model:** `text-embedding-3-small` (OpenAI)
- **Số chunks:** 268 chunks
- **Chunk size:** 1000 ký tự, overlap 200
- **Retrieval:** Top-5 chunks với filter theo checker type

**7 tài liệu ISO gốc:**
- ISO 128-1:2020 — Nguyên tắc trình bày chung
- ISO 128-2:2020 — Quy ước đường nét
- ISO 128-3:2020 — Hướng nhìn, mặt cắt
- ISO 129-1:2018 — Ghi kích thước và dung sai
- ISO/DIS 129-2 — Dung sai hình dạng và vị trí
- ISO 1101:2017 — Dung sai hình học (GD&T)
- ISO 7200:2004 — Khung tên bản vẽ kỹ thuật

**Trạng thái:** Hoạt động ổn định, tự động index lần đầu khởi động

---

### 3.2. MySQL (Relational Database) 

**Mục đích:** Lưu trữ users, sessions, analysis history, logs, quota

**Cấu hình:**
- **Database:** `drawing_checker`
- **Charset:** `utf8mb4_unicode_ci` (hỗ trợ tiếng Việt và emoji)
- **Engine:** InnoDB
- **Connection:** SQLAlchemy async với `asyncmy` driver
- **Pool size:** 5–20 connections
- **Pool recycle:** 3600 seconds

**Schema hiện tại (5 bảng):**

#### 1. `users` — Quản lý tài khoản người dùng
| Field | Type | Mô tả |
|-------|------|-------|
| `id` | BIGINT UNSIGNED | Primary key |
| `email` | VARCHAR(255) | Email đăng nhập (unique) |
| `password_hash` | VARCHAR(255) | Mật khẩu đã hash (NULL nếu OAuth) |
| `display_name` | VARCHAR(100) | Tên hiển thị |
| `role` | ENUM('user', 'admin') | Vai trò |
| `avatar_url` | VARCHAR(500) | URL ảnh đại diện |
| `is_active` | BOOLEAN | Trạng thái tài khoản |
| `oauth_provider` | ENUM('google', 'github') | Nhà cung cấp OAuth |
| `oauth_id` | VARCHAR(255) | ID từ OAuth provider |
| `created_at` | DATETIME | Ngày tạo |
| `updated_at` | DATETIME | Ngày cập nhật |

**Indexes:** email, role, (oauth_provider, oauth_id)

#### 2. `quota_usage` — Theo dõi giới hạn phân tích
| Field | Type | Mô tả |
|-------|------|-------|
| `id` | BIGINT UNSIGNED | Primary key |
| `user_id` | BIGINT UNSIGNED | Foreign key → users |
| `usage_date` | DATE | Ngày sử dụng (UTC+7) |
| `request_count` | INT UNSIGNED | Số lần đã dùng |
| `max_requests` | INT UNSIGNED | Giới hạn tối đa (mặc định: 10) |
| `updated_at` | DATETIME | Ngày cập nhật |

**Unique key:** (user_id, usage_date)

#### 3. `analysis_history` — Lịch sử phân tích bản vẽ
| Field | Type | Mô tả |
|-------|------|-------|
| `id` | BIGINT UNSIGNED | Primary key |
| `user_id` | BIGINT UNSIGNED | Foreign key → users |
| `task_id` | VARCHAR(36) | UUID v4 (unique) |
| `filename` | VARCHAR(255) | Tên file DXF |
| `drawing_type` | VARCHAR(100) | Loại bản vẽ |
| `status` | ENUM | pending/running/completed/failed |
| `high_errors` | INT UNSIGNED | Số lỗi nghiêm trọng |
| `medium_errors` | INT UNSIGNED | Số lỗi trung bình |
| `low_errors` | INT UNSIGNED | Số lỗi nhẹ |
| `report_content` | LONGTEXT | Báo cáo đầy đủ (JSON/Markdown) |
| `error_message` | TEXT | Thông báo lỗi (nếu failed) |
| `created_at` | DATETIME | Ngày tạo |

**Indexes:** user_id, task_id, status

#### 4. `activity_logs` — Log hoạt động hệ thống
| Field | Type | Mô tả |
|-------|------|-------|
| `id` | BIGINT UNSIGNED | Primary key |
| `user_id` | BIGINT UNSIGNED | Foreign key → users (NULL cho system events) |
| `event_type` | VARCHAR(50) | Loại sự kiện |
| `ip_address` | VARCHAR(45) | IPv4/IPv6 |
| `details` | JSON | Chi tiết sự kiện |
| `severity` | ENUM('INFO', 'WARNING', 'ERROR') | Mức độ |
| `created_at` | DATETIME | Thời gian |

**Event types:** login_success, login_fail, analysis_start, analysis_complete, analysis_error, quota_exceeded, etc.

**Indexes:** user_id, event_type, severity, created_at

#### 5. `sessions` — Quản lý phiên đăng nhập
| Field | Type | Mô tả |
|-------|------|-------|
| `id` | BIGINT UNSIGNED | Primary key |
| `user_id` | BIGINT UNSIGNED | Foreign key → users |
| `jwt_token_hash` | VARCHAR(64) | SHA-256 hash của JWT (unique) |
| `expires_at` | DATETIME | Thời gian hết hạn |
| `created_at` | DATETIME | Ngày tạo |

**Indexes:** user_id, expires_at

**Trạng thái:** Schema đã được định nghĩa, đang trong giai đoạn migration và integration với FastAPI backend

## 4. Hiệu Suất

| Bước | Thời gian |
|------|-----------|
| Parse DXF + Render | 2–5s |
| RAG Indexing (lần đầu) | 30–90s |
| Analysis Node | 15–30s |
| 3 Checkers | 45–75s |
| Self-Check Node | 0–60s |
| **Tổng** | **~1.5–3 phút** |
